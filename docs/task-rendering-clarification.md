# Task 渲染机制澄清

## 用户反馈的关键点

### 1. Task 列表是主要渲染内容

**Main Agent**：
- 显示完整的工具输出
- 显示 Task 列表（当调用 TaskList 时）
- 显示详细的执行过程

**Subagent**：
- **主要显示 Task 列表**（清晰展示主线任务流程）
- 工具输出被简化/截断
- 目的：让用户聚焦在任务进度，而不是淹没在详细输出中

### 2. 工具输出的渐进式更新

**不是一次性输出全部内容**，而是：

```
╭── ✦ Read  src/auth.ts ────────────────────────────────╮
│  Reading file... (1024 bytes)                         │  ← 第一行：状态
╰───────────────────────────────────────────────────────╯

↓ 更新为

╭── ✦ Read  src/auth.ts ────────────────────────────────╮
│  Lines 1-50 of 200                                    │  ← 第二行：进度
╰───────────────────────────────────────────────────────╯

↓ 完成后

╭── ✔ Read  src/auth.ts ────────────────────────────────╮
│  200 lines read                                       │  ← 最终：总结
╰───────────────────────────────────────────────────────╯
```

**关键特性**：
- 只显示几行（通常 1-3 行）
- 滚动更新（原地更新同一个工具块）
- 完成后显示总结信息，而不是完整输出

### 3. Task 列表的实时更新

```
Main Agent 创建任务：
╭── ✔ TaskCreate ───────────────────────────────────────╮
│  Created task #1: Analyze codebase                    │
│  Created task #2: Write documentation                 │
│  Created task #3: Run tests                           │
╰───────────────────────────────────────────────────────╯

Subagent 执行时，StatusBar 显示：
┌────────────────────────────────────────────────────────┐
│ Codara │ haiku │ Task #1 [in_progress] │ ⣾           │
└────────────────────────────────────────────────────────┘

Subagent 完成后，Main Agent 看到：
╭── ✔ TaskUpdate ───────────────────────────────────────╮
│  Task #1 completed by subagent-explore                │
╰───────────────────────────────────────────────────────╯
```

## 正确的 TUI 渲染机制

### Main Agent 视图

```
> 请分析代码库并生成文档

我来帮你完成这个任务。

╭── ✔ TaskCreate ───────────────────────────────────────╮
│  Created 3 tasks                                      │
╰───────────────────────────────────────────────────────╯

╭── ✦ Agent  explore-agent ────────────────────────────╮
│  Spawning subagent...                                 │
╰───────────────────────────────────────────────────────╯

[等待 subagent 完成...]

╭── ✔ Agent  explore-agent ────────────────────────────╮
│  Completed: Found 42 source files, 15 components     │
╰───────────────────────────────────────────────────────╯

╭── ✔ TaskList ─────────────────────────────────────────╮
│  #1 [completed] Analyze codebase (owner: explore)    │
│  #2 [in_progress] Write documentation (owner: main)  │
│  #3 [pending] Run tests                              │
╰───────────────────────────────────────────────────────╯
```

### Subagent 视图（简化）

```
[Subagent: explore-agent]

╭── ✔ TaskList ─────────────────────────────────────────╮
│  #1 [in_progress] Analyze codebase (owner: me)       │
│  #2 [pending] Write documentation                    │
│  #3 [pending] Run tests                              │
╰───────────────────────────────────────────────────────╯

╭── ✦ Glob  src/**/*.ts ────────────────────────────────╮
│  Found 42 files                                       │  ← 只显示总结
╰───────────────────────────────────────────────────────╯

╭── ✦ Read  src/index.ts ───────────────────────────────╮
│  Reading... 150 lines                                 │  ← 只显示总结
╰───────────────────────────────────────────────────────╯

╭── ✔ TaskUpdate ───────────────────────────────────────╮
│  Task #1 marked as completed                          │
╰───────────────────────────────────────────────────────╯
```

**关键差异**：
- Main Agent：显示详细输出
- Subagent：只显示总结信息 + Task 列表
- 目的：让用户清晰看到任务流程，而不是被细节淹没
